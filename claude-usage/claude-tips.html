Quick Tips
  • MODELLOUsa Sonnet al posto di Opus: 80% dei task, 5x meno costoso
  • TIMINGTrick: manda un prompt alle 6:00, lavora dalle 9:00. Reset alle 11:00
  • CTXTorna al modello 200K se vedi "1M" o "extended" nel nome
  • TASKUna conversazione per task, non lasciare crescere i thread
  • PROMPTScrivi "fix line 42 in auth.ts" non "fix auth" (10x piu' economico)
  • BATCH3 domande in un prompt invece di 3 follow-up separati
  • CTXEsegui /compact spesso, CLAUDE.md sotto 60 righe
  • FILEAggiungi .claudeignore per escludere dir irrilevanti
  • BUGSessione gia' consumata all'avvio: apri nuova conversazione
  • AUTOProva le routine di Claude: /cron in Claude Code per task schedulati automatici (reminder, log, cleanup)
CLI Config - risparmio 60-80% in un colpo
{ "model": "sonnet", "env": { "MAX_THINKING_TOKENS": "10000", "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50", "CLAUDE_CODE_SUBAGENT_MODEL": "haiku" } }
  • FILESalva in ~/.claude/settings.json
  • THINKMAX_THINKING_TOKENS: da 32K default a 10K (~70% risparmio)
  • CTXAutocompact al 50% invece di 95%: sessioni piu' sane
  • EXTRA.claudeignore + CLAUDE.md sotto 60 righe (90% meno costo avvio)

Bottom Line per Piano
Pro $20/mo
  • Appena distinguibile dal Free in questo momento
  • Solo sezione A (disciplina d'uso), non espandibile
  • CLI config obbligatoria per reggere
  • Sessioni che partono consumate: bug noto, riapri
  • Trick timing sessione: manda un prompt alle 6:00
Max $100-200/mo
  • settings.json + hook + CLAUDE.md snello = 3-5x piu' utilizzo
  • Peak multiplier attivo comunque
  • Combinato con Codex/Cursor e' il setup piu' efficiente
  • Penalizzante per chi usa i default (la maggioranza)
Web e Mobile
